# Design Decisions

## 1. Purpose

这个文件用于记录重要设计决策，避免以后忘记为什么这样设计。

| TBD | 建立独立 docs 文档体系 | 防止项目后期混乱 | 后续改代码前应先更新文档 |
| TBD | 子应用必须模块化 | 方便长期扩展和维护 | 新增应用时不会污染其他模块 |

## 3. Change Rule

以后如果网站格局发生变化，必须遵守：

1. 不要先改代码。
2. 先更新相关 docs 文档。
3. 在本文件新增一条设计决策记录。
4. 写清楚为什么改。
5. 写清楚影响哪些页面、权限、路由、子应用和数据。
6. 再根据文档修改代码。
7. 修改后补充测试和验收说明。
