# Public Site Design

## 1. Purpose

公开访客网站是所有访客默认看到的页面。

它用于展示个人形象、项目、作品、文章或其他公开内容。具体展示内容当前尚未完全确定，未确定部分必须标记为 TBD。

## 2. Confirmed Requirements

- 打开域名后默认进入公开访客网站。
- 公开网站不应该像后台系统。
- 公开网站应该像正常个人主页。
- 页面中有可见但低调的入口按钮。
- `访客入口` 进入公开 journey 原型页。
- `用户入口` 进入登录占位页。

## 3. Visible Entrance Pattern

首页入口的当前设计方向是可见但低调的导航按钮：

- `访客入口` 位于首页左下角，进入 `journey.html`。
- `用户入口` 位于首页左下角同一组入口中，进入 `login.html`。
- 入口应保持安静、紧凑，不应变成大型应用启动器。
- 入口按钮不能遮挡 ICP footer 或主要 hero 内容。

未来如果更换视觉表达，也应保持入口可见、可理解、低干扰。

首页入口只是从公开主页进入其他页面的视觉入口，不是安全措施。

当前静态阶段，用户入口链接到 `login.html` 登录占位页。该占位页不提供真实登录、认证或权限能力。
